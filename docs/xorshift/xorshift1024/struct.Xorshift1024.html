A random number generator that uses the xorshift1024* algorithm [1].

Description

Quoted from [1].

This is a fast, top-quality generator. If 1024 bits of state are too much, try a xoroshiro128+ generator.

Note that the three lowest bits of this generator are LSFRs, and thus they are slightly less random than the other bits. We suggest to use a sign test to extract a random Boolean value.

The state must be seeded so that it is not everywhere zero. If you have a 64-bit seed, we suggest to seed a splitmix64 generator and use its output to fill s.

[1]: Sebastiano Vigna, xorshift1024*

Parallelism

The RngJump implementation is equivalent to 2512 calls to next_u64(). Used to generate 2512 non-overlapping subsequences for parallel computations.
